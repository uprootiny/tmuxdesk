features #

mesh protocol

Session state propagates via fire-and-forget SSH. Status bar reads local files only. No polling, no central server, no database.

sigil addressing

Each node gets a Unicode sigil from alchemy, math, or the I Ching. Not decoration — a fleet-wide addressing scheme you read at a glance.

cross-fleet search

f searches sessions + panes across all nodes. G finds git repos fleet-wide with activity, branches, Claude Code history.

self-deploying

One command rsyncs config to all nodes, writes ~/.tmux.conf, hot-reloads. --verify checks the whole fleet.

core / extensions

Base config needs only tmux + bash. Extensions (fzf, fleet ops, TPM) load from ext.conf — delete it for minimal.

threshold-balanced mesh

Push 30s → fresh 60s → stale 120s → prune 300s. The chain is documented and balanced against heartbeat interval.

keybindings #

f fuzzy jump — sessions, panes, fleet G project search across all nodes F fleet health dashboard B broadcast command to all nodes P preset layout picker S create or switch to named session Tab cycle layout forward M-l cycle layout backward h j k l navigate panes | - split horizontal / vertical r reload config m toggle mouse

install #

setup 
# clone
$ git clone https://github.com/uprootiny/tmuxdesk
$ cd tmuxdesk

# edit fleet.conf with your nodes, then:
$ ./deploy.sh

# start the heartbeat (each node)
$ ~/.tmux/tmuxdesk/bin/mesh-heartbeat-loop.sh &

# verify fleet health
$ ./deploy.sh --verify

Requires tmux 3.2+, bash 4+. Optional: fzf for fuzzy features, TPM for session persistence.
